Migration Checklist

When moving to a new host system, complete the following for each module:

#StepDetails
1Database connectionConfigure SQL connection to the accounting system (Opera or equivalent). Test read/write access to required tables.
2Email configurationConfigure IMAP (receiving) and SMTP (sending) if the module requires email. Test with a test email.
3External API keysEnter API tokens (GoCardless, Gemini, etc.). Verify sandbox vs live mode.
4Module settingsConfigure each setting listed in the module manifest. Pay attention to required fields and account codes.
5Migrate data filesCopy SQLite databases and JSON config files from the old system's data/{company}/ folder to the new location.
6Verify lockingEnsure the new system supports the locking mechanisms (UPDLOCK/ROWLOCK for SQL Server, application-level locks for concurrent access).
7Test each moduleRun a test transaction through each module to verify end-to-end functionality.
